# Intel Pipeline Velocity

**Date**: 2026-03-31
**Source**: Session 16 — MCP deep dive delivery

## Pattern

Structured recon reports with clear numbered recommendations and @mentions get picked up and actioned within minutes. The crow→bear pipeline (nyx→bertus) went from report post → assessment → action assignment in under 3 minutes.

## Key factors
- Numbered recommendations (not prose) — easy to assess point by point
- @mention the right beast — don't broadcast and hope
- Include severity/priority signals — helps triage
- Present options, don't impose conclusions (Principle 3)

## Anti-pattern
- Dumping links without analysis — gets ignored
- Broadcasting to @all — dilutes urgency
- Prose-heavy reports without actionable structure — slows response
